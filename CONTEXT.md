# Gist

Gist 帮助用户建立和阅读自己的订阅内容资料库。桌面客户端既能独立使用，也能使用已经部署在其他设备上的 Gist 服务。

## Language

**桌面客户端**：
可安装在桌面系统上的 Gist 应用，支持在**本地模式**与**远程模式**之间选择。
_Avoid_: Wails 客户端、远程客户端

**本地模式**：
桌面客户端在当前设备上独立提供完整 Gist 能力的运行模式，不要求另行部署服务，也不向桌面客户端之外共享本地资料库。
_Avoid_: 离线模式、单机壳

**远程模式**：
桌面客户端使用一套已经单独部署的 Gist 服务及其内容资料库的运行模式。
_Avoid_: Web 模式、云模式

**内容资料库**：
一组归属于同一 Gist 使用环境的订阅、文章、阅读状态、收藏和设置。本地模式与远程模式的内容资料库相互独立。
_Avoid_: 数据库、实例数据

**网络代理设置**：
归属于一个内容资料库、用于控制承载该资料库的 Gist 服务访问外部资源的设置。桌面客户端按当前模式直接读写对应服务的配置，不在本地与远程之间复制。
_Avoid_: 代理同步、共享代理、桌面连接代理

**本地资料库**：
桌面客户端在本地模式中使用的唯一内容资料库。一个桌面客户端不管理多个本地资料库。
_Avoid_: 本地工作区、本地配置

**远程连接**：
桌面客户端保存的、用于进入远程模式的一套 Gist 服务连接。一个桌面客户端最多保存一个；移除连接只删除本机记录，不删除远程资料库。
_Avoid_: 远程资料库、远程账号

**远程服务地址**：
用户手动提供并经过验证的 Gist 服务基址。它使用 HTTP 或 HTTPS，可以带有反向代理路径，但不包含登录凭据、查询参数或片段。
_Avoid_: API 端点、远程网页地址

**本地用户**：
只属于本地资料库的单一用户身份。首次进入本地模式时创建，并使用本地凭据登录。
_Avoid_: 桌面账号、设备账号

**远程用户**：
由远程连接所指向的 Gist 服务识别的用户身份。它与本地用户没有身份或凭据继承关系。
_Avoid_: 云账号、本地用户

**模式切换**：
桌面客户端改变当前所使用内容资料库的操作。模式切换不复制、迁移或同步内容，也不暂停本地资料库既有的后台维护。
_Avoid_: 资料库同步、数据切换

**启动模式**：
桌面客户端上次使用并将在下次启动时优先进入的运行模式。首次启动尚无启动模式，必须由用户选择。
_Avoid_: 默认资料库、自动模式

**远程不可用**：
远程连接暂时无法提供其内容资料库的状态。桌面客户端不保存可替代它的离线资料库，并保持远程模式，直到用户重试、修改连接或显式切换模式。
_Avoid_: 本地回退、自动离线模式

**本地完整备份**：
由用户主动保存、通过独立密码保护并用于完整恢复一个本地资料库的快照。它包含本地账号和本地敏感设置，不包含远程连接或远程资料库。
_Avoid_: OPML 备份、远程备份

**完整恢复**：
使用本地完整备份整体替换当前本地资料库的操作。完整恢复不是资料合并，失败时保留恢复前的本地资料库。
_Avoid_: 导入、合并恢复

**前台任务**：
由用户主动发起、其进度或结果当前仍需要用户关注的资料库操作。存在前台任务时，模式切换必须先由用户决定取消还是留下。
_Avoid_: 后台维护、定时刷新

## Flagged ambiguities

- “客户端”曾可能仅指连接远程服务的界面；现已明确为同时支持**本地模式**和**远程模式**的**桌面客户端**。
- “切换模式”仅表示切换当前**内容资料库**，不能用来表示资料库之间的迁移或同步。
- “同步代理配置”应表述为按当前模式读写**网络代理设置**；本地与远程配置始终各自归属，不形成同步副本。
- “远程模式”不表示支持多个服务器；首版最多保存一个**远程连接**。
- “移除远程连接”不能用来表示注销、删除远程用户或删除远程资料库。
- “本地运行”只描述**本地模式**，不表示桌面客户端同时成为可供浏览器或其他设备访问的服务端。
- “登录”必须带有所属模式；**本地用户**与**远程用户**的凭据和登录状态不能混用。
- “记住模式”只设置**启动模式**，不代表自动迁移资料库或共享登录状态。
- “远程不可用”不能触发隐式**模式切换**，否则用户可能误操作另一套内容资料库。
- “远程不可用”也不表示进入远程离线副本；首版不存在这样的内容资料库。
- “当前使用远程模式”只描述当前资料库，不表示本地资料库已被关闭或暂停维护。
- “备份”必须区分**本地完整备份**与 OPML；二者保护的数据范围不同。
- **本地完整备份**的密码不属于任何本地用户或远程用户，应用不会保存或找回它。
- “恢复备份”统一指**完整恢复**；不能用来描述向当前资料库追加或合并数据。
- **前台任务**与资料库后台维护不同；只有前台任务会阻止直接模式切换。

## Example dialogue

> 用户：我在笔记本上没有部署服务器，可以直接用桌面客户端吗？
> 开发者：可以，选择本地模式会建立当前设备的内容资料库。
>
> 用户：我已经在家里的服务器上使用 Gist，桌面客户端能打开那里的订阅吗？
> 开发者：可以，选择远程模式后使用那套服务的内容资料库。
>
> 用户：我在本地模式中读完文章，远程模式中也会自动变成已读吗？
> 开发者：不会；两个内容资料库相互独立，模式切换不会同步状态。
>
> 用户：我可以同时保存家庭和公司的两个远程服务吗？
> 开发者：首版不可以；桌面客户端最多保存一个远程连接。
>
> 用户：移除远程连接会删掉服务器上的订阅吗？
> 开发者：不会；它只清除桌面客户端保存的连接信息。
>
> 用户：我已经登录远程模式，切换到本地模式还要登录吗？
> 开发者：要；本地用户与远程用户相互独立，各自完成注册或登录。
>
> 用户：下次打开桌面客户端还要重新选择模式吗？
> 开发者：不用；它会优先进入上次使用的启动模式，你仍可在应用内显式切换。
>
> 用户：远程服务离线时，客户端会自动打开本地资料库吗？
> 开发者：不会；它会停留在远程不可用页面，由你重试、修改连接或显式切换模式。
>
> 用户：我在远程模式里阅读时，本地资料库还会更新吗？
> 开发者：会；只要桌面客户端仍在运行，本地资料库会继续后台维护。
>
> 用户：导出 OPML 就备份了我的已读和收藏状态吗？
> 开发者：没有；要保存本地资料库的完整状态，需要创建本地完整备份。
>
> 用户：完整恢复会把备份内容与当前文章合并吗？
> 开发者：不会；它会在确认和回滚保护下整体替换当前本地资料库。
>
> 用户：AI 摘要还在生成时可以直接切到远程模式吗？
> 开发者：客户端会先让你取消该前台任务或留在当前模式，不会悄悄切换。
